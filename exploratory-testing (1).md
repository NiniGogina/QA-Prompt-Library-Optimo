# Prompt Name
Exploratory Testing Guide Prompt — Optimo

## Purpose
ეს პრომტი ეხმარება QA ინჟინერს Optimo-ს მოდულების სისტემატური Exploratory Testing-ის ჩასატარებლად.
გამოიყენება მაშინ, როცა სპეციფიკაცია სრული არ არის ან ახალი ფუნქციონალობის სწრაფ შემოწმებასთან გვაქვს საქმე.

---

## Prompt

```
შენ ხარ გამოცდილი QA ინჟინერი, რომელიც ატარებს Exploratory Testing-ს Optimo-ს სისტემაში.

Optimo არის ფილიალების მართვის სისტემა შემდეგი მოდულებით:
- გაყიდვები და შესყიდვები
- მარაგები
- მომწოდებლებთან ანგარიშსწორება
- იურიდიულ პირზე გაყიდვები
- წარმოება

შექმენი Exploratory Testing Charter შემდეგი მოდულისთვის:

მოდული: [ᲛᲝᲓᲣᲚᲘᲡ ᲡᲐᲮᲔᲚᲘ]
ტესტირების დრო: [მაგ. 60 წუთი]
ტესტერის გამოცდილება: [Junior / Middle / Senior]

Charter-ი უნდა შეიცავდეს:
1. მისია (რა ვამოწმებთ)
2. ტესტირების სფეროები (Areas to Explore)
3. რისკ-ზონები (სად ყველაზე მეტი ბაგი მოსალოდნელია)
4. კითხვები, რომლებზეც პასუხი გვჭირდება
5. იდეები ტესტ-სცენარებისთვის

გამომავალი ფორმატი: სტრუქტურირებული Markdown
```

## მაგალითი გამოყენებისა

**Input:**
> მოდული: მარაგები | დრო: 60 წუთი | Senior

**Output (ნიმუში):**

### 🎯 მისია
გამოვავლინოთ ბაგები Optimo-ს მარაგების მოდულში, განსაკუთრებით ფილიალებს შორის გადაადგილების, სასაქონლო ნაშთების გამოთვლისა და ნებართვების სფეროში.

### 🔍 ტესტირების სფეროები
- პროდუქტების სია და ფილტრაცია
- მარაგის შევსება (Receipt)
- ფილიალებს შორის გადაცემა (Transfer)
- ნაშთების ანგარიში (Inventory Report)
- ლიმიტების და გაფრთხილებების სისტემა

### ⚠️ რისკ-ზონები
- ერთდროული ოპერაციები ორი ფილიალიდან (Race Condition)
- მარაგი რიცხვი ნულზე ქვემოთ (Negative Stock)
- ძალიან დიდი ოდენობების შეყვანა
- პერმიშენი: კასირმა ნახოს/ვერ ნახოს სხვა ფილიალის მარაგი

### ❓ კითხვები
- რა ხდება, თუ ერთი პროდუქტი ორ ფილიალში ერთდროულად განახლდა?
- ნეგატიური მარაგი დაიშვება?
- მარაგის ცვლილების ჟურნალი (Audit Log) ინახება?

### 💡 ტესტ-სცენარების იდეები
1. Transfer მოთხოვნა + ერთდროულად Cancel
2. 0.001 ოდენობით პროდუქტის დამატება
3. პროდუქტი 1000+ სიმბოლოიანი სახელით
4. სხვადასხვა ფილიალის მომხმარებელი — ერთი პროდუქტი
