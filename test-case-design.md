# Prompt Name
Test Case Design Prompt — Optimo

## Purpose
ეს პრომტი გამოიყენება Optimo-ს ნებისმიერი მოდულისთვის ფუნქციონალური სატესტო კეისების გენერაციისთვის.
მოიცავს: გაყიდვები, შესყიდვები, მარაგები, მომწოდებლებთან ანგარიშსწორება, იურიდიულ პირზე გაყიდვები, წარმოება.

---

## Prompt

```
შენ ხარ სინიორ QA ინჟინერი, რომელიც ტესტავს Optimo-ს — ფილიალების მართვის სისტემას.

სისტემის მოდულებია:
- გაყიდვები და შესყიდვები
- მარაგები
- მომწოდებლებთან ანგარიშსწორება
- იურიდიულ პირზე გაყიდვები
- წარმოება

გაანალიზე ქვემოთ მოცემული ფუნქციონალობის აღწერა და დაგენერირე სრულყოფილი სატესტო კეისები.

ყურადღება გაამახვილე:
- პოზიტიური სცენარები (Happy Path)
- ნეგატიური სცენარები (ვალიდაცია, შეცდომები)
- საზღვრული მნიშვნელობები (Edge Cases)
- ნებართვები და როლები (ადმინი vs კასირი vs მენეჯერი)
- ფილიალებს შორის მონაცემთა კავშირი

მოდული: [ᲛᲝᲓᲣᲚᲘᲡ ᲡᲐᲮᲔᲚᲘ]
ფუნქციონალობის აღწერა:
[ჩასვი ფუნქციონალობის აღწერა]

გამომავალი ფორმატი:
| TC ID | სცენარი | წინაპირობა | ნაბიჯები | მოსალოდნელი შედეგი | პრიორიტეტი |
```

**მოდული:** გაყიდვები
**Input:**
> მომხმარებელმა უნდა შეძლოს ახალი გაყიდვის დოკუმენტის შექმნა, პროდუქტების დამატება ოდენობითა და ფასით, და გადახდის დასრულება.

**Output (ნიმუში):**

| TC ID | სცენარი | წინაპირობა | ნაბიჯები | მოსალოდნელი შედეგი | პრიორიტეტი |
|-------|---------|-----------|---------|-------------------|-----------|
| TC-S-001 | წარმატებული გაყიდვის შექმნა | მომხმარებელი ავტორიზებულია, პროდუქტი მარაგშია | 1. გახსენი გაყიდვების მოდული 2. დააჭირე "ახალი გაყიდვა" 3. დაამატე პროდუქტი 4. შეიყვანე ოდენობა 5. დაასრულე გადახდა | დოკუმენტი შეინახება, მარაგი შემცირდება | High |
| TC-S-002 | გაყიდვა 0 ოდენობით | მომხმარებელი ავტორიზებულია | 1. ახალი გაყიდვა 2. ოდენობა: 0 3. შეინახე | სისტემამ უნდა აჩვენოს ვალიდაციის შეცდომა | High |
| TC-S-003 | გაყიდვა — მარაგი ამოიწურა | პროდუქტის მარაგი = 0 | 1. ახალი გაყიდვა 2. ოდენობა: 1 3. შეინახე | შეტყობინება: "მარაგი არ არის" | Medium |

## Tips
- TC ID-ში გამოიყენე მოდულის პრეფიქსი: `TC-S` (Sales), `TC-P` (Purchase), `TC-I` (Inventory), `TC-W` (Warehouse), `TC-L` (Legal), `TC-M` (Manufacturing)
- პრიორიტეტი: High / Medium / Low
- ყოველი კეისი ერთ კონკრეტულ ქცევას ამოწმებს
