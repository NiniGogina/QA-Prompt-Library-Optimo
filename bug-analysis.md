# Prompt Name
Bug Analysis & Report Prompt — Optimo

## Purpose
ეს პრომტი ეხმარება QA ინჟინერს Optimo-ში ნაპოვნი ბაგების სრულყოფილი ანალიზის, კლასიფიკაციისა და ანგარიშის შედგენაში.
გამოიყენება: ბაგის ანგარიშის დასაწერად, Root Cause-ის ასახსნელად და Dev-ისთვის გასაგზავნად.

---

## Prompt A — ბაგის ანგარიშის გენერაცია

```
შენ ხარ სინიორ QA ინჟინერი Optimo-ს პროექტზე.

Optimo-ს მოდულებია: გაყიდვები, შესყიდვები, მარაგები, მომწოდებლებთან ანგარიშსწორება, იურიდიულ პირზე გაყიდვები, წარმოება.

ქვემოთ მოცემულ ბაგის აღწერის საფუძველზე შეადგინე სრული Bug Report.

ბაგის აღწერა:
[ჩასვი ბაგის მოკლე აღწერა]

Bug Report-ი უნდა შეიცავდეს:
- სათაური (Title) — კონკრეტული და ინფორმაციული
- მოდული / კომპონენტი
- სიმძიმე (Severity): Critical / Major / Minor / Trivial
- პრიორიტეტი (Priority): High / Medium / Low
- გამეორების ნაბიჯები (Steps to Reproduce)
- ფაქტობრივი შედეგი (Actual Result)
- მოსალოდნელი შედეგი (Expected Result)
- გარემო (Environment): Browser, OS, Optimo ვერსია
- თანდართული მასალა: [Screenshot / Log]
- შენიშვნა განმავითარებლისთვის
```

## Prompt B — Root Cause Analysis

```
შენ ხარ QA Lead Optimo-ს პროექტზე.

გაანალიზე ქვემოთ მოცემული ბაგი და შეადგინე Root Cause Analysis (RCA).

ბაგის სათაური: [ᲑᲐᲒᲘᲡ ᲡᲐᲗᲐᲣᲠᲘ]
ბაგის კონტექსტი: [ᲙᲝᲜᲢᲔᲥᲡᲢᲘ]

RCA-ი უნდა შეიცავდეს:
1. პრობლემის მოკლე აღწერა
2. პირდაპირი მიზეზი (Immediate Cause)
3. ძირეული მიზეზი (Root Cause) — "5 Whys" მეთოდით
4. გავლენა (Impact) — ვის და რას ეხება
5. პრევენციის გეგმა (Prevention Plan)
6. სარეკომენდაციო ფიქსი
```

## მაგალითი — Bug Report

**Input:**
> გაყიდვების მოდულში, როდესაც ვქმნი ახალ გაყიდვას და ვამატებ პროდუქტს, სისტემა ვერ ახდენს მარაგის განახლებას.

**Output (ნიმუში):**

---

**სათაური:** [SALES-042] გაყიდვის დოკუმენტის შენახვის შემდეგ მარაგი არ მცირდება

**მოდული:** გაყიდვები → მარაგებთან ინტეგრაცია

**Severity:** Critical
**Priority:** High

**გამეორების ნაბიჯები:**
1. Optimo-ში შედი გაყიდვების მოდულში
2. დააჭირე "ახალი გაყიდვა"
3. დაამატე პროდუქტი "ყავა 1კგ", ოდენობა: 2
4. დაასრულე გაყიდვა — "შენახვა"
5. გადადი მარაგებში — შეამოწმე "ყავა 1კგ"-ის ნაშთი

**ფაქტობრივი შედეგი:**
ნაშთი არ შეცვლილა — ისე დარჩა 50 ერთეული

**მოსალოდნელი შედეგი:**
ნაშთი უნდა შემცირდეს: 50 → 48

**გარემო:**
- Browser: Chrome 124
- OS: Windows 11
- Optimo Version: 2.4.1

**შენიშვნა Dev-ისთვის:**
სავარაუდოდ პრობლემა გაყიდვისა და მარაგის სერვისებს შორის event trigger-ში. შეამოწმე `sale.confirmed` event handler.
